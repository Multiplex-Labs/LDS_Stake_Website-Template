---
title: "Building Reservations"
description: "Public reservation submission and admin approval endpoints."
---

The reservations router provides a public form-submission endpoint for members and organizations to request building use, and a set of admin endpoints for stake leaders to review, approve, deny, or delete requests.

Base path: `/reservations`

## Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/reservations/` | None | Submit a building reservation request |
| `GET` | `/reservations/` | `APPROVE_BLDG_RESERVATIONS` | List reservation requests |
| `GET` | `/reservations/{reservation_id}` | `APPROVE_BLDG_RESERVATIONS` | Get a single reservation |
| `POST` | `/reservations/{reservation_id}/approve` | `APPROVE_BLDG_RESERVATIONS` | Approve a reservation |
| `POST` | `/reservations/{reservation_id}/deny` | `APPROVE_BLDG_RESERVATIONS` | Deny a reservation with a reason |
| `DELETE` | `/reservations/{reservation_id}` | `APPROVE_BLDG_RESERVATIONS` | Delete a reservation record |

---

## POST /reservations/

Submits a new building reservation request. No authentication is required — this is a public endpoint accessible from the `/stake-info/reserve` page.

The system checks for scheduling conflicts against existing approved reservations. A successful response always returns `201`, but the body includes a `conflict` flag when overlapping reservations were detected.

**Request body:**

```json
{
  "event_name": "Young Women Camp Meeting",
  "event_description": "Monthly coordination meeting for Young Women leaders.",
  "date": "2026-08-15",
  "start_time": "18:00",
  "end_time": "20:00",
  "setup_time": "17:30",
  "cleanup_time": "20:30",
  "rooms": ["Cultural Hall", "Kitchen"],
  "organizer_name": "Sister Johnson",
  "organizer_email": "johnson@example.com",
  "organizer_phone": "801-555-0200",
  "organization": "Young Women",
  "organization_other": null,
  "affiliation": "LDS",
  "needs_access": true
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event_name` | string | Yes | Short name for the event |
| `event_description` | string | Yes | Full description of the event |
| `date` | date (ISO 8601) | Yes | Date of the reservation |
| `start_time` | time (`HH:MM`) | Yes | Event start time |
| `end_time` | time (`HH:MM`) | Yes | Event end time |
| `setup_time` | time (`HH:MM`) | No | When setup access is needed |
| `cleanup_time` | time (`HH:MM`) | No | When cleanup will be finished |
| `rooms` | list[string] | Yes | Rooms requested (at least one) |
| `organizer_name` | string | Yes | Full name of the person responsible |
| `organizer_email` | string | Yes | Contact email for notifications |
| `organizer_phone` | string | Yes | Contact phone number |
| `organization` | string | Yes | Organization name (e.g., "Young Women", "Other") |
| `organization_other` | string | No | Required when `organization` is `"Other"` |
| `affiliation` | string | Yes | Affiliation declaration |
| `needs_access` | boolean | Yes | Whether the organizer needs a building access code |

**Response 201:**

```json
{
  "id": 42,
  "status": "pending",
  "conflict": false,
  "event_name": "Young Women Camp Meeting",
  ...
}
```

When `conflict` is `true`, the reservation has been saved but an admin will need to resolve the scheduling overlap before approving.

## GET /reservations/

Returns a list of reservation requests. Supports optional filtering.

**Query parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by status: `pending`, `approved`, `denied` |
| `date` | date (ISO 8601) | Filter to a specific date |

**Response 200:** An array of reservation objects sorted by date.

---

## GET /reservations/\{reservation_id\}

Returns a single reservation by its ID, including full organizer details and room list.

**Response 404:** Reservation not found.

---

## POST /reservations/\{reservation_id\}/approve

Marks a reservation as approved. Sends a notification email to the organizer.

**Response 200:** The updated reservation object with `status: "approved"`.

**Response 409 Conflict:** Cannot approve a reservation that is already denied or deleted.

---

## POST /reservations/\{reservation_id\}/deny

Marks a reservation as denied and records the reason. Sends a notification email to the organizer including the reason text.

**Request body:**

```json
{ "reason": "The building is unavailable on that date due to a stake conference." }
```

**Response 200:** The updated reservation object with `status: "denied"`.

---

## DELETE /reservations/\{reservation_id\}

Permanently removes the reservation record. Use this for housekeeping after a reservation has been processed, or to remove erroneous submissions.

**Response 204:** No content.

<Callout type="info">
  The `APPROVE_BLDG_RESERVATIONS` permission is a separate bitmask flag in the permissions system. It is not automatically granted by other admin permissions. Assign it explicitly to the users or callings responsible for building scheduling.
</Callout>
