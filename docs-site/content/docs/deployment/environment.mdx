Each service reads its configuration from a `.env` file in its own directory. None of the services share environment files.

## Backend

File location: `backend/.env`

| Variable | Required | Default | Description |
|---|---|---|---|
| `DEV` | Optional | `false` | Enables debug mode and exposes the Swagger UI at `/docs`. Set to `false` in production. |
| `PORT` | Optional | `8000` | Port the FastAPI server listens on. |
| `DB_ENGINE` | Optional | `sqlite` | Database engine. Accepted values: `sqlite`, `postgresql`. |
| `DATABASE_PATH` | Optional | `./database.db` | Path to the SQLite database file. Only used when `DB_ENGINE=sqlite`. |
| `INITIAL_ADMIN_PASSWORD` | Required (first run only) | ŌĆö | Seeds the `admin@admin.com` account when the database is empty. Ignored on all subsequent starts. |
| `JWT_SECRET_KEY` | Required | ŌĆö | Secret used to sign and verify JWT tokens. Use a long random string in production ŌĆö generate one with `openssl rand -hex 32`. |
| `SSL_ENABLED` | Optional | `false` | Set to `true` when the application is served over HTTPS. Marks the `refresh_token` cookie as `Secure`. |
| `SP_APPROVAL_THRESHOLD` | Optional | `2` | Number of Stake Presidency approvals required to advance a calling proposal from `SP_APPROVAL` to `HC_APPROVAL`. |
| `HC_APPROVAL_THRESHOLD` | Optional | `3` | Number of High Council approvals required to advance a calling proposal from `HC_APPROVAL` to `INTERVIEW`. |

### Email

Both providers are optional. When both are configured, Brevo is used first and Mailjet serves as the fallback. Omitting both disables outbound email silently.

| Variable | Required | Default | Description |
|---|---|---|---|
| `BREVO_API_KEY` | Optional | ŌĆö | Brevo transactional API key. When set, Brevo is used as the primary email provider (300 emails/day on the free tier). |
| `BREVO_FROM_EMAIL` | Optional | ŌĆö | Sender address for Brevo. Required when `BREVO_API_KEY` is set. |
| `MAILJET_API_KEY` | Optional | ŌĆö | Mailjet API key. Used as the email fallback when Brevo is not configured. |
| `MAILJET_SECRET_KEY` | Optional | ŌĆö | Mailjet secret key. Required when `MAILJET_API_KEY` is set. |
| `MAILJET_FROM_EMAIL` | Optional | ŌĆö | Sender address for Mailjet. Required when `MAILJET_API_KEY` is set. |
| `REPLY_TO_EMAIL` | Optional | ŌĆö | Reply-to address shown to email recipients. Recommended: a monitored clerk inbox (e.g. `clerk@yourstake.org`). Falls back to `BREVO_FROM_EMAIL` if unset. |

### Google Calendar

Both variables are required together. Omitting either disables Google Calendar sync silently.

| Variable | Required | Default | Description |
|---|---|---|---|
| `GOOGLE_SERVICE_ACCOUNT_FILE` | Optional | ŌĆö | Filesystem path to a GCP service account JSON file. Enables Google Calendar sync for appointment bookings. |
| `GOOGLE_CALENDAR_ID` | Optional | ŌĆö | Google Calendar ID (e.g. `abc123@group.calendar.google.com`). Required together with `GOOGLE_SERVICE_ACCOUNT_FILE`. |

### CORS and URLs

| Variable | Required | Default | Description |
|---|---|---|---|
| `ALLOWED_ORIGINS` | Optional | `http://localhost:3100` | Comma-separated list of allowed frontend origins for CORS. In production, set this to your public site URL. |
| `FRONTEND_BASE_URL` | Optional | `http://localhost:3100` | Base URL used in redirect responses and email links pointing to the frontend. |
| `BACKEND_BASE_URL` | Optional | `http://localhost:8000` | Base URL used in email links pointing directly to the backend (e.g. email confirmation links). |

<Callout type="warn">
  Never commit `JWT_SECRET_KEY` or `INITIAL_ADMIN_PASSWORD` to version control. Add `backend/.env` to `.gitignore`.
</Callout>

## Frontend

File location: `frontend/.env`

| Variable | Required | Default | Description |
|---|---|---|---|
| `PORT` | Optional | `3100` | Port the Express proxy server listens on. |
| `NODE_ENV` | Optional | `development` | Set to `production` for production builds and deployments. Controls minification and other optimizations. |
| `BACKEND_URL` | Optional | `http://localhost:8000` | URL the Express proxy forwards all `/api/*` requests to. In Docker Compose, use the internal service name (e.g., `http://backend:8000`). |

<Callout type="info">
  The frontend does not require a `.env` file for local development ŌĆö defaults point to `localhost:8000`, which is where the backend runs.
</Callout>

## Discord Bot

File location: `discordbot/.env`

The Discord bot is a standalone service that connects to Discord via the gateway API and also exposes a FastAPI HTTP server on port 8001. It communicates with the main backend using the credentials below.

| Variable | Required | Default | Description |
|---|---|---|---|
| `DISCORD_TOKEN` | Required | ŌĆö | Discord bot token from the Discord developer portal (under Bot ŌåÆ Token). |
| `BACKEND_URL` | Required | ŌĆö | URL of the main FastAPI backend (e.g. `http://localhost:8000`). The Discord bot calls this to look up users and notify approvers. |
| `BACKEND_TOKEN` | Required | ŌĆö | Service-to-service authentication token used when the Discord bot calls the main backend API. Must match the token configured in the main backend. |
| `DATABASE_PATH` | Optional | `./database.db` | Path to the Discord bot's own SQLite database file (used for `UserMapping` and audit logs). |
| `DEV` | Optional | `false` | Enables debug logging and Uvicorn reload mode. Set to `false` in production. |

<Callout type="warn">
  Never commit `DISCORD_TOKEN` or `BACKEND_TOKEN` to version control. Add `discordbot/.env` to `.gitignore`.
</Callout>
