---
title: "Environment Variables"
description: "Complete reference for all configuration values across the backend, frontend, and Slackbot services."
---

Each service reads its configuration from a `.env` file in its own directory. None of the services share environment files.

## Backend

File location: `backend/.env`

| Variable | Required | Default | Description |
|---|---|---|---|
| `DEV` | Optional | `false` | Enables debug mode and exposes the Swagger UI at `/docs`. Set to `false` in production. |
| `PORT` | Optional | `8000` | Port the FastAPI server listens on. |
| `DB_ENGINE` | Optional | `sqlite` | Database engine. Accepted values: `sqlite`, `postgresql`. |
| `DATABASE_PATH` | Optional | `./database.db` | Path to the SQLite database file. Only used when `DB_ENGINE=sqlite`. |
| `INITIAL_ADMIN_PASSWORD` | Required (first run only) | — | Seeds the `admin@admin.com` account when the database is empty. Ignored on all subsequent starts. |
| `JWT_SECRET_KEY` | Required | — | Secret used to sign and verify JWT tokens. Use a long random string in production — generate one with `openssl rand -hex 32`. |
| `SSL_ENABLED` | Optional | `false` | Set to `true` when the application is served over HTTPS. Marks the `refresh_token` cookie as `Secure`. |
| `SP_APPROVAL_THRESHOLD` | Optional | `2` | Number of Stake Presidency approvals required to advance a calling proposal from `SP_APPROVAL` to `HC_APPROVAL`. |
| `HC_APPROVAL_THRESHOLD` | Optional | `3` | Number of High Council approvals required to advance a calling proposal from `HC_APPROVAL` to `INTERVIEW`. |

<Callout type="warn">
  Never commit `JWT_SECRET_KEY` or `INITIAL_ADMIN_PASSWORD` to version control. Add `backend/.env` to `.gitignore`.
</Callout>

## Frontend

File location: `frontend/.env`

| Variable | Required | Default | Description |
|---|---|---|---|
| `PORT` | Optional | `3100` | Port the Express proxy server listens on. |
| `NODE_ENV` | Optional | `development` | Set to `production` for production builds and deployments. Controls minification and other optimizations. |
| `BACKEND_URL` | Optional | `http://localhost:8000` | URL the Express proxy forwards all `/api/*` requests to. In Docker Compose, use the internal service name (e.g., `http://backend:8000`). |

<Callout type="info">
  The frontend does not require a `.env` file for local development — defaults point to `localhost:8000`, which is where the backend runs.
</Callout>

## Slackbot

File location: `slackbot/.env`

| Variable | Required | Default | Description |
|---|---|---|---|
| `SLACK_BOT_TOKEN` | Required | — | Bot OAuth token from your Slack app (starts with `xoxb-`). Found in the Slack app dashboard under **OAuth & Permissions**. |
| `SLACK_APP_TOKEN` | Required | — | App-level token from your Slack app (starts with `xapp-`). Required for Socket Mode. Found under **Basic Information → App-Level Tokens**. |

<Callout type="info">
  Socket Mode does not require a public inbound URL or webhook endpoint. The Slackbot establishes an outbound WebSocket connection to Slack, making it suitable for deployments behind NAT or a firewall.
</Callout>
