---
title: "Docker Compose Deployment"
description: "Deploy all three Stake Portal services with a single Docker Compose command."
---

Docker Compose is the recommended deployment method. It starts the frontend/proxy, backend, and (optionally) the Slackbot as isolated containers with a single command.

## Step 1: Clone the Repository

```bash
git clone <your-repo-url>
cd LDS_Stake_Website-Template-2
```

## Step 2: Create the Backend Environment File

Create `backend/.env` with the following variables:

```bash
# backend/.env
DEV=false
PORT=8000
DB_ENGINE=sqlite
DATABASE_PATH=./database.db
INITIAL_ADMIN_PASSWORD=change-me-on-first-run
JWT_SECRET_KEY=replace-with-a-long-random-string
SSL_ENABLED=false
SP_APPROVAL_THRESHOLD=2
HC_APPROVAL_THRESHOLD=3
```

<Callout type="warn">
  `JWT_SECRET_KEY` must be a long, random string in production. Anyone who knows this value can forge authentication tokens. Generate one with `openssl rand -hex 32`.
</Callout>

See [Environment Variables](/deployment/environment) for a full description of each setting.

## Step 3: Create the Frontend Environment File

Create `frontend/.env`:

```bash
# frontend/.env
PORT=3100
NODE_ENV=production
BACKEND_URL=http://backend:8000
```

<Callout type="info">
  In Docker Compose, the frontend proxy reaches the backend via the internal service name `backend`, not `localhost`. If you rename the service in your `docker-compose.yml`, update `BACKEND_URL` to match.
</Callout>

## Step 4: Create the Slackbot Environment File (Optional)

If you want the Slackbot service to run, create `slackbot/.env`:

```bash
# slackbot/.env
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_APP_TOKEN=xapp-your-app-level-token
```

If you do not need the Slackbot, you can comment out or remove the `slackbot` service from `docker-compose.yml` before starting.

## Step 5: Start All Services

```bash
docker compose up -d
```

This starts:

| Container | Description | Port |
|---|---|---|
| `frontend` | React SPA + Express proxy | 3100 |
| `backend` | FastAPI REST API | 8000 (internal) |
| `slackbot` | Slack Bolt (Socket Mode) | None (outbound only) |

## Step 6: Verify the Deployment

```bash
curl http://localhost:3100/api/health
```

A healthy response looks like:

```json
{"status": "ok"}
```

The portal UI is available at [http://localhost:3100](http://localhost:3100).

## First Login

<Callout type="info">
  Before logging in for the first time, read [First Run & Bootstrap](/deployment/first-run). The admin account is created automatically on first start, and you will be required to change the password on first login.
</Callout>

## Viewing Logs

```bash
# All services
docker compose logs -f

# Backend only
docker compose logs -f backend

# Frontend only
docker compose logs -f frontend
```

## Stopping Services

```bash
docker compose down
```

To also remove named volumes (database, etc.):

```bash
docker compose down -v
```
