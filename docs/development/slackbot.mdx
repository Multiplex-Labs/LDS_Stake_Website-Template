The slackbot is a standalone Python service that connects to Slack via **Socket Mode**. It uses the [Slack Bolt for Python](https://slack.dev/bolt-python/) framework and runs entirely independently of the frontend and backend.

<Info>
  Socket Mode means the bot connects **outbound** to Slack over a persistent WebSocket. No inbound webhook URL, no public IP, and no firewall rules are required. This makes it easy to run locally or behind NAT.
</Info>

---

## Required environment variables

Create a `.env` file inside `slackbot/` with these two variables:

```bash
SLACK_BOT_TOKEN=xoxb-...    # Bot User OAuth Token
SLACK_APP_TOKEN=xapp-...    # App-Level Token (for Socket Mode)
```

<Warning>
  Never commit these tokens to source control. Add `slackbot/.env` to `.gitignore` if it is not already there.
</Warning>

## Creating a Slack app

<Steps>
  <Step title="Create the app">
    Go to [https://api.slack.com/apps](https://api.slack.com/apps) and click **Create New App → From scratch**. Give it a name and select your workspace.
  </Step>
  <Step title="Enable Socket Mode and generate the App-Level Token">
    In the left sidebar, click **Socket Mode** and toggle it on. You will be prompted to generate an **App-Level Token**. Give it a name and add the `connections:write` scope. Copy the token — it starts with `xapp-`. This is your `SLACK_APP_TOKEN`.
  </Step>
  <Step title="Add bot token scopes">
    In the left sidebar, go to **OAuth & Permissions → Bot Token Scopes** and add the following scopes:

    | Scope | Purpose |
    |-------|---------|
    | `app_mentions:read` | Receive events when the bot is @-mentioned |
    | `chat:write` | Send messages |
    | `im:history` | Read direct message history |
    | `im:read` | Read DM channel info |
    | `im:write` | Open DM channels |
  </Step>
  <Step title="Install the app to your workspace">
    In the left sidebar, go to **OAuth & Permissions** and click **Install to Workspace**. After installing, copy the **Bot User OAuth Token** — it starts with `xoxb-`. This is your `SLACK_BOT_TOKEN`.
  </Step>
  <Step title="Enable the messages.im event">
    In the left sidebar, go to **Event Subscriptions**, enable events, and subscribe to the `message.im` bot event. This allows the bot to receive direct messages.
  </Step>
</Steps>

## Starting the bot

```bash
cd slackbot
uv sync          # Install dependencies (first time only)
uv run python main.py
```

A successful start will print a connection confirmation to stdout. If you see a token error, double-check that both environment variables are set correctly in `slackbot/.env`.

---

## Current feature state

The slackbot is in an early stage. What currently works:

- The bot connects to Slack via Socket Mode successfully.
- When a user sends a direct message containing the word **"hello"**, the bot replies with a greeting.

What is not yet built:

- No integration with the FastAPI backend — the bot runs fully standalone.
- No stake-specific commands or lookups.
- No event handling beyond the basic "hello" response.

Future integrations with FastAPI (e.g., looking up assignments, querying the speaking schedule) are planned as future work.
